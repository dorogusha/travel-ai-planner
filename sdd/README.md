# Specification-Driven Development (SDD)

This folder stores product and engineering artifacts that drive implementation.

## Structure

- `specs/` - problem statement, goals, requirements, and acceptance criteria.
- `design/` - architecture, data flow, and implementation decisions.
- `tasks/` - executable implementation checklist linked to specs and design.

## Workflow

1. Write or update a spec in `specs/`.
2. Draft the implementation design in `design/`.
3. Break work into small tasks in `tasks/`.
4. Implement and keep all three documents aligned.
